    filtering Parameters : {
        "keep_ratio": 
        "filter_function": the used filering function (see under this)
        "selection_config": {
            min_brightness_value: minimum brightness value for selection of best frames, float value between 0.0 to 100.0
            max_brightness_value: maximum brightness value for selection of best frames, float value between min_brightness_value to 100.0
            brightness_step: step value for iterating over brightness values, float value between 0.0 to max_brightness_value-min_brightness_value
            min_entropy_value: minimum entropy value for selection of best frames, float value between 0.0 to 10.0
            max_entropy_value: maximum entropy value for selection of best frames, float value between min_entropy_value to 10.0
            entropy_step: step value for iterating over entropy values, float value between 0.0 to max_entropy_value-min_entropy_value
        } (only affect the brightness_contrast_movement_based_selecting function)
    }

    filtering functions :{
        uniform_sampling : {
            simple uniform downsampling
            it doesn't use the selection_config 
        },

        brightness_contrast_movement_based_selecting : {
            
            An Iterative Sharpness-Diversity Filter that uses statistical clustering and information entropy to select a representative 
            subset of high-contrast, non-blurry frames from video frames.
            
            ### The 4-Stage Logic Flow

            1.  Quality Pruning (Brightness & Entropy)
                Using the `ImageSelectionConfig`, the function filters frames based on their Information Density. 
                * Brightness Range: Discards frames that are too dark (underexposed) or too bright (overexposed/screen glare), 
                which is critical for detecting Replay Attacks.
                * Entropy (Contrast): Measures texture richness. Frames with low entropy (blurry or flat surfaces) are removed.
                * Iterative Relaxation: If initial requirements are too strict, it uses the `brightness_step` and `entropy_step` 
                to gradually relax thresholds until enough frames are found.

            2.  Visual Diversity (Clustering)
                To solve the "Redundancy Problem," it uses K-Means Clustering on grayscale histograms.
                * It groups visually similar frames together (e.g., all frames with a specific head angle or background).
                * This ensures that the final selection represents the entire duration of the video rather than just one specific second.
                

            3.  Sharpness Optimization (Laplacian Variance)
                Within each visual cluster, the function calculates the Variance of the Laplacian. 
                * This is a focus-measure tool used to identify the sharpest frame in a group.
                * By selecting only the crispest frames, you ensure the CNN can see the fine details 
                (skin pores, edges) necessary for high-accuracy spoof detection.
            
            

            ### Summary Table 

            | Component         | Parameter Used        | Goal |
        
            | Exposure Filter   | `min/max_brightness`  | Remove "blind" or glare-heavy frames. |
            | Texture Filter    | `min/max_entropy`     | Ensure high information density. |
            | Diversity Engine  | `nb_clusters`         | Prevent overfitting to redundant frames. |
            | Focus Check       | `Laplacian Variance`  | Select the sharpest candidate in each group. |


        }
    }
